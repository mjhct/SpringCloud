##2. 问题与解决
###2.1 请求与响应数据格式
application.yml进行配置
```yaml
spring:
  jackson:
    # 过滤空值
    default-property-inclusion: non_null
    # 驼峰与下划线转换
    property-naming-strategy: CAMEL_CASE_TO_LOWER_CASE_WITH_UNDERSCORES
```
###2.2 入驻Eureka响应数据变成XML
Controller请求映射指定响应数据类型
```
@GetMapping(value = "/test", produces = MediaType.APPLICATION_JSON_VALUE)
```

###2.3 Jmeter调接口响应乱码
Controller请求映射指定响应数据类型及编码
```
// PPLICATION_JSON_UTF8_VALUE被废弃的是因为主流浏览器已经符合规范正确解释UTF-8编码了，所以没必要带上Charset了
@PostMapping(value = "/test", produces = MediaType.APPLICATION_JSON_UTF8_VALUE)
```
###2.4 Hystrix图形化监控报错
这个SpringCloud版本默认的依赖2.2.3.RELEASE版本有BUG。  
+ [参考链接1](https://github.com/Netflix-Skunkworks/hystrix-dashboard/issues/11)
+ [参考链接2](https://blog.csdn.net/oyc619491800/article/details/107420176)  

其实起决定性作用的是这个依赖包里面依赖的jquery，所以把它排除掉用别的版本就好了；好像`2.2.3.RELEASE`后依赖的jquery都是`3.4.1`，`2.2.2.RELEASE`依赖的还是`2.1.1`。
```
<dependency>
    <groupId>org.springframework.cloud</groupId>
    <artifactId>spring-cloud-starter-netflix-hystrix-dashboard</artifactId>
    <exclusions>
        <exclusion>
            <artifactId>spring-cloud-netflix-hystrix-dashboard</artifactId>
            <groupId>org.springframework.cloud</groupId>
        </exclusion>
    </exclusions>
</dependency>
<dependency>
    <groupId>org.springframework.cloud</groupId>
    <artifactId>spring-cloud-netflix-hystrix-dashboard</artifactId>
    <!-- 现在最新2.2.4.RELEASE版本会报别的错，用2.2.2.RELEASE就是正常的 -->
    <version>2.2.2.RELEASE</version>
</dependency>
```
